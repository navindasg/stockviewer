# Task 2: SQLite Database & Migrations

**PRD Reference:** Sections 3 (Data Model), 7.1 (SQLite Schema), 7.2 (IPC API)

## Description

Set up `better-sqlite3` in the Electron main process. Create the database file, run the schema migrations (transactions, price_cache, ticker_metadata tables), and expose basic CRUD operations via IPC.

## Subtasks

- [x] Install `better-sqlite3` with exact version pinning
- [x] Create `electron/db/database.ts` — SQLite setup, connection, and migration runner
- [x] Implement the `transactions` table with all columns and constraints from Section 7.1
- [x] Implement the `price_cache` table with composite primary key
- [x] Implement the `ticker_metadata` table
- [x] Create indexes on `transactions(ticker)` and `transactions(date)`
- [x] Create `electron/db/positions.ts` — transaction CRUD (add, update, delete, list with filters)
- [x] Create `electron/db/priceCache.ts` — price cache read/write operations
- [x] Wire up IPC handlers in `main.ts` for all CRUD operations
- [x] Expose typed API surface via `preload.ts` (matching Section 7.2 interface)
- [x] Generate UUIDs for transaction IDs

## Acceptance Criteria

- [x] Database file is created on first launch in the app's user data directory
- [x] All three tables exist with correct schema, constraints, and indexes
- [x] `addTransaction` creates a record and returns it with generated `id`, `created_at`, `updated_at`
- [x] `updateTransaction` modifies a record and updates `updated_at`
- [x] `deleteTransaction` removes a record by ID
- [x] `getTransactions` supports filtering by ticker
- [x] SELL validation: rejects sells that exceed held shares at that point in time (chronological check)
- [x] `shares > 0` and `price > 0` constraints are enforced
- [x] IPC round-trip works from renderer → main → SQLite → main → renderer
